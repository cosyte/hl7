## Overview

The v2.1 milestone delivers `@cosyte/hl7` 0.1.0 — the inaugural public release of Cosyte's HL7 v2 toolkit for Node.js/TypeScript. The library realises the project's north star: *a developer can parse a real-world, vendor-quirky HL7 v2 message and pull useful fields out of it in one line — without having read the HL7 spec.*

Phases 1–9 covered the v1 functional surface (parser, model, helpers, serialization, profiles, tests, examples, docs, package rename). Phases 10–12 closed v2.1-MILESTONE-AUDIT tech debt: planning-doc resync, retroactive `VERIFICATION.md` for Phases 1/8/9, and retroactive Nyquist `VALIDATION.md` for Phases 1/2/3/7/8/9. The end state is a paper trail that matches the code ground-truth — 9/9 verified, 9/9 Nyquist-validated, 97/97 v1 REQ-IDs closed.

**Key Decisions:**
- **Postel's Law applied at the API boundary:** parser is liberal (lenient by default + warnings), serializer is conservative (always emits spec-clean HL7). Strict mode escalates Tier-2 deviations to thrown errors.
- **4-tier deviation model:** silent / warn / fatal / strict-only. 4 fatal codes (Tier-3) are the only thrown-by-default conditions; everything else is a warning with a stable code + position + snippet.
- **Zero runtime dependencies:** Node stdlib only. Dual ESM + CJS via `tsup`. ES2022 / Node 18+.
- **Profile system as a first-class public API:** `defineProfile()` with `extends`/merge semantics + 5 built-in vendor profiles (epic, cerner, meditech, athena, genericLab). Starter kit (`examples/profile-starter-kit/`) ships publishable as-is.
- **Package rename mid-milestone (`@cosyte/hl7-parser` → `@cosyte/hl7`, Phase 9):** name reflects the actual surface area — a full HL7 toolkit (parser, builder, mutator, serializer, helpers). CHANGELOG carries a D-07 rename breadcrumb.
- **Coverage gate scenario A (Phase 7):** per-dir branches 85→90 on parser/model/helpers/serialize/builder; global kept at 85 to avoid implicitly gating ungated `profiles/**`. CI `Test (with coverage)` step runs across Node 18/20/22.
- **Audit follow-on as separate phases (10/11/12) rather than retroactive edits:** v2.1-MILESTONE-AUDIT (status: tech_debt) drove discrete planning-doc resync, retroactive `VERIFICATION.md`, and retroactive Nyquist `VALIDATION.md` phases. Each gap-closure phase has its own plans + summaries + verifier report — full GSD traceability rather than untracked patch commits.
