    Note: `YYYY-MM-DDTHH:mm:ss` contains literal `T` — NOT a token — so validateDateFormats needs to pass (the `T` is a separator literal per the format-matcher contract). This is the ISO-style format the BUILTIN_DATE_FALLBACKS ISO-8601 path ALREADY handles via `parseIso8601`, but declaring it in the profile short-circuits the "tried second-to-last" fallback ordering by moving it to the FIRST user-format try. Net effect: ISO-8601-T is found in the user-format loop, `TIMESTAMP_FALLBACK_FORMAT` still emits naming it (normal behavior per TOL-08), but the PROFILE authoritatively owns it. (Plan 06 fixture tests assert warning reduction COUNT, not absence — the specific dynamics are in scope for Plan 06's test design.)
